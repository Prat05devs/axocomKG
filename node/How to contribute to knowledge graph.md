## Copy the Knowledge Graph Locally
### Step 1
Install Git in your local computer. 
https://docs.github.com/en/desktop/installing-and-authenticating-to-github-desktop/installing-github-desktop

### Step 2.
Login to the application. if you don't have an account create an account on [github](https://www.github.com)

### Step 3
- In GitHub Desktop, go to **Clone a repository**.
- Paste the following URL into the repository URL field:
   `https://github.com/basictech01/axocomKG`
- Choose the appropriate local path where you want the repository to be stored.
- Click **Clone**.

### Step 4 
Navigate to the local path you selected in Step 3 and open the **axocomKG** folder using **Obsidian**.

## Contributing to the Knowledge Graph

### Step 1: Sync with the Main Branch

1. In GitHub Desktop, click **Fetch origin** (top-right corner).
2. If updates are available, click **Pull origin** to get the latest changes.

### Step 2: Create a New Branch

1. Click on the **Current Branch** dropdown (top center).
2. Create a new branch with an appropriate, descriptive name.
3. Ensure the branch is created from the **main** branch.

### Step 3: Make Your Changes

Use Obsidian to make your changes. This may include:

- Adding new nodes
- Editing or updating existing nodes

### Step 4: Commit Your Changes

1. In GitHub Desktop, review the changes you made.
2. In the bottom-left panel, fill in:
    - **Summary**: A short, clear description of the change
    - **Description**: A detailed explanation of what you changed and why
3. Click **Commit**.

### Step 5: Publish the Branch

Click **Publish branch** in the top-right corner to push your branch to the remote repository.

### Step 6: Create a Pull Request

1. Click **Create Pull Request** in GitHub Desktop.
2. This will open GitHub in your browser.
3. Click the green **Create pull request** button to submit your PR.

### Step 7: Request a Review

Ask relevant team members to review your pull request.

## Contribution Guidelines & Precautions

1. **One change per PR/commit**
    - Each pull request or commit should modify only one document.
    - If you need to change multiple documents, create separate PRs/commits.

2. **Write clear commit messages**    
    - Use meaningful commit summaries.
    - Clearly describe what you changed and why in the commit description.
    - Avoid vague or empty messages.
